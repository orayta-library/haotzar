# טיפים לשיפור ביצועים - האויצר

## 🚀 בעיות ביצועים נפוצות ופתרונות

### 1. **אינדקס חיפוש כבד**
**בעיה:** האינדקס נטען בכל הפעלה ולוקח זמן רב
**פתרון:** ✅ טעינה עצלה - האינדקס נטען רק בעת חיפוש

### 2. **חיפוש אוטומטי מיותר**
**בעיה:** חיפוש בכל הקלדה בסרגל העליון
**פתרון:** ✅ Debounce של 300ms

### 3. **קבצים רבים מדי**
**בעיה:** טעינת אלפי קבצים בבת אחת
**פתרון:** 
- הגבל את מספר הקבצים המוצגים
- השתמש ב-virtual scrolling לרשימות ארוכות

### 4. **עיבוד PDF איטי**
**בעיה:** חילוץ טקסט מ-PDF לוקח זמן רב
**פתרון:**
- בנה אינדקס מראש עם הסקריפט: `node scripts/tools.js index:build`
- השתמש ב-Meilisearch במקום FlexSearch

### 5. **זיכרון גבוה**
**בעיה:** האפליקציה צורכת הרבה זיכרון
**פתרון:**
- סגור כרטיסיות שלא בשימוש
- נקה cache של PDF-ים ישנים

## 🔧 פקודות אופטימיזציה

### בניית אינדקס מהיר:
```bash
# בניית אינדקס FlexSearch
node scripts/tools.js index:build

# בניית אינדקס Meilisearch (מהיר יותר)
node scripts/tools.js index:build
```

### בדיקת ביצועים:
```bash
# בדיקת סטטוס Meilisearch
node scripts/tools.js meili:status

# בדיקת אינדקס
node scripts/tools.js index:check
```

## 📊 מדדי ביצועים טובים

- **זמן הפעלה:** < 3 שניות
- **זמן חיפוש:** < 500ms
- **זיכרון:** < 500MB
- **גודל אינדקס:** < 100MB

## ⚠️ סימני אזהרה

אם האפליקציה איטית, בדוק:
1. כמה קבצים יש בתיקיית הספרים
2. האם האינדקס נבנה (קובץ `flexsearch-index.json`)
3. האם Meilisearch פועל (`http://localhost:7700`)
4. האם יש שגיאות בקונסול

## 🛠️ פתרון בעיות נפוצות

### "האפליקציה תקועה בטעינה"
```bash
# מחק אינדקס ישן
rm -f flexsearch-index.json
# בנה מחדש
node scripts/tools.js index:build
```

### "חיפוש לא עובד"
```bash
# בדוק סטטוס
node scripts/tools.js meili:status
# הפעל מחדש
node scripts/tools.js index:build
```

### "זיכרון גבוה"
- סגור כרטיסיות לא נחוצות
- הפעל מחדש את האפליקציה
- הקטן את מספר הקבצים בתיקייה
